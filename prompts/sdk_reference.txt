n8n Workflow SDK — Cheat Sheet (validado contra validate_workflow del MCP server real)

REGLA 1 (LA MÁS IMPORTANTE): NO escribas NINGUNA sentencia `import`. Las funciones del SDK ya están
disponibles globalmente. Una línea `import ... from '@n8n/workflow-sdk'` HACE FALLAR la validación
("Import declarations are not allowed in SDK code"). Funciones disponibles sin importar:
workflow, node, trigger, ifElse, switchCase, merge, splitInBatches, nextBatch, newCredential, expr.

REGLA 2: NO uses nombres reservados (workflow, node, trigger, ifElse, expr, ...) como nombre de
variable. MAL: `const workflow = workflow(...)`. BIEN: nombrá las variables distinto y exportá con
`export default workflow('id', 'Nombre')...`.

REGLA 3: Definí cada nodo/trigger como una const aparte, y encadenalos SOLO en el `export default
workflow(...)`. No anides definiciones de nodos dentro de la config de otro. Nada de arrow functions.

REGLA 4: Credenciales. Si la credencial YA EXISTE (aparece en CREDENCIALES DISPONIBLES con su `id`),
referenciala por **ID exacto**: `newCredential('Nombre', 'id-exacto-de-la-lista')` — así el nodo queda
linkeado a la credencial real. `newCredential('Nombre')` SIN id **NO linkea** (deja el nodo sin auth);
usalo solo si no hay ninguna credencial disponible para ese tipo. Nunca inventes IDs ni claves.

Webhook Trigger (v2.1):
const webhook = trigger({
  type: 'n8n-nodes-base.webhook',
  version: 2.1,
  config: {
    name: 'Webhook Trigger',
    parameters: { path: 'webhook-path', httpMethod: 'POST', responseMode: 'onReceived' }
  }
});

Filtro condicional / ifElse (v2.2) — usá expr() para expresiones n8n:
const checkError = ifElse({
  version: 2.2,
  config: {
    name: 'Check Error',
    parameters: {
      conditions: {
        options: { caseSensitive: false, typeValidation: 'loose' },
        conditions: [
          {
            leftValue: expr('{{ $json.body?.message ?? $json.message }}'),
            operator: { type: 'string', operation: 'contains' },
            rightValue: 'error'
          }
        ],
        combinator: 'and'
      }
    }
  }
});

Nodo Gmail (v2.1):
const sendEmail = node({
  type: 'n8n-nodes-base.gmail',
  version: 2.1,
  config: {
    name: 'Send Gmail',
    parameters: {
      resource: 'message',
      operation: 'send',
      sendTo: 'user@example.com',
      subject: 'Error Alert',
      messageType: 'text',
      message: expr('Error: {{ $json.body?.message ?? $json.message }}')
    },
    credentials: { gmailOAuth2: newCredential('Gmail OAuth2', 'ID_EXACTO_DE_CREDENCIALES_DISPONIBLES') }
  }
});

Composición (encadenar SOLO acá; .onTrue() para la rama verdadera del ifElse):
export default workflow('webhook-filter-gmail', 'Webhook Filter Gmail Workflow')
  .add(webhook)
  .to(checkError.onTrue(sendEmail));
